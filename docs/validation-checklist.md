# Validation Checklist

Use this checklist to validate Geo Market Watch outputs before sharing or acting on them.

## Pre-Analysis Checklist

- [ ] One or more news links provided
- [ ] Links are accessible and relevant to geopolitical event
- [ ] If links missing, confidence reduction explicitly stated

## Source Authentication Checklist

- [ ] All sources classified into Tier 1 / Tier 2 / Low-trust
- [ ] Tier 1 sources used only for Confirmed Facts
- [ ] Official rhetoric distinguished from confirmed impact
- [ ] Low-trust sources downranked or ignored
- [ ] Fog of War situations flagged (`仍存在争议`)

## Content Quality Checklist

### Confirmed Facts Section
- [ ] Only Tier 1 sources used
- [ ] Source tags present (`[官方]`, `[通讯社]`, etc.)
- [ ] Dates, quantities, physical nodes specified
- [ ] Contested facts remain contested, not harmonized

### Market Interpretation Section
- [ ] Macro backdrop (rates, inflation, growth) in 2-3 sentences
- [ ] Transmission path clearly mapped
- [ ] What market is trading, not just news summary

### Scenario Analysis Section
- [ ] A/B/C scenarios covered (or better framework)
- [ ] Each scenario has trigger condition
- [ ] Each scenario has market impact
- [ ] Switch conditions specified

### Key Indicators Section
- [ ] Indicators are publicly verifiable
- [ ] Data gaps explicitly noted (`数据滞后/缺失`)
- [ ] Alternative proxies suggested when needed

### Watchlist Section
- [ ] US / A-share / Hong Kong markets covered
- [ ] Aggressive / Balanced / Defensive buckets
- [ ] Each security has mapping logic
- [ ] Each security has trigger signal
- [ ] Each security has invalidation condition
- [ ] Hedges labeled clearly

### Commentary Section
- [ ] Most important variable identified
- [ ] Potential over/under-estimation noted
- [ ] Scenario-breaking conditions specified

### Risk Warning Section
- [ ] Fact vs interpretation vs scenario clearly separated
- [ ] Framework-breaking variables identified

## Output Format Checklist

- [ ] All 9 sections present in order
- [ ] Bilingual labels correct (`[已确认事实 / Confirmed Facts]`)
- [ ] Tables properly formatted
- [ ] No absolute trading instructions
- [ ] Uncertainty made visible

## Final Quality Gate

- [ ] Would I act on this if it were my money?
- [ ] Are the invalidation conditions specific enough to actually use?
- [ ] If key data is missing, is that clearly stated?
- [ ] Does this read like analysis or marketing?
