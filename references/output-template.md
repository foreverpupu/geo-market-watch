# 输出模板

严格按以下顺序输出，不要改节名。
关键中文标题首次出现时保留英文注释，降低外部工具误编码后的歧义。

## 1. 事件快照
- 用 2 到 4 句话说明发生了什么、何时发生、为什么重要。

## 2. 已确认事实 / Confirmed Facts
- 只能写由 Tier 1 信源锚定的事实。
- 每条关键事实都加来源级别标签，例如：
  - [官方]
  - [通讯社]
  - [公司文件]
  - [交易所/ETF发行方]
  - [官方机构/航运通告]
- 尽量写清：日期、量化指标、物理节点。
- 如果关键事实在可信来源之间存在冲突，明确写：**仍存在争议**。
- 不要把争议事实收敛成单一确定结论；必要时简要列出相互冲突的说法，并说明后续该看什么来验证。

## 3. 市场解读 / Market Interpretation
- 使用 Tier 2 结构分析和最新核验信息。
- 先用不超过 2 到 3 句话写宏观底色：利率方向、通胀背景、增长敏感性。
- 再写主传导路径：事件 -> 商品 / 物流 / 政策 / 情绪 -> 板块 -> 观察池。
- 明确市场当前主要在交易什么，而不是泛泛复述新闻。

## 4. 情景推演 / Scenario Analysis
- 至少覆盖 A / B / C 三种情景，除非事件有更优框架。
- 每种情景都要说明：
  - 触发条件
  - 可能的市场影响
  - 从当前基准情景切换过去需要看到什么

## 5. 关键观察指标
| 指标 | 为什么重要 | 当前读数/状态 | 对观察池的含义 |
|---|---|---|---|
|  |  |  |  |

要求：
- 优先使用公开、可验证、与事件直接相关的指标。
- 如果关键高频数据拿不到，明确写：**数据滞后/缺失**，并改用公开 proxy。

## 6. 三地观察池
| 市场 | 分组 | 代码/名称 | 映射逻辑 | 触发信号 | 失效条件 | 属性 |
|---|---|---|---|---|---|---|
| 美股 / A股 / 港股 | 进攻型 / 平衡型 / 防守型 |  |  |  |  | 受益 / 对冲 |

要求：
- 三地默认都覆盖，但允许按传导路径不对称展开。
- 如果某标的是对冲而非直接受益，要明确标注。

## 7. 分组总结
### 进攻型
- 目标为 5 只；若高质量直接映射标的不够，则给 3 到 4 只，并注明：**（未强行补足）**

### 平衡型
- 目标为 5 只；若高质量直接映射标的不够，则给 3 到 4 只，并注明：**（未强行补足）**

### 防守型
- 目标为 5 只；若高质量直接映射标的不够，则给 3 到 4 只，并注明：**（未强行补足）**

## 8. 短评
用 2 到 4 段短评说明：
- 当前最重要的变量是什么
- 市场可能高估或低估了什么
- 什么情况会推翻当前基准情景

## 9. 风险提示
- 明确哪些判断来自已确认事实。
- 明确哪些判断来自市场解读。
- 明确哪些判断来自情景推演。
- 明确最可能推翻当前框架的变量。
